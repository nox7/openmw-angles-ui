Common Pitfalls with Loops and Signals

Because we are dealing with signals and for loops, that means we're dealing with a table of data. There may be a time where you mutate an existing table and expect the UI to update when you set your signal value to the same table. This will not work, as the signal will not detect a change in value since the reference is the same. To get around this, you can create a new table and copy the values over, then set your signal to that new table. This will trigger the UI to update as the reference has changed.
