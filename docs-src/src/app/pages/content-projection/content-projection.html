Content Projection

When you utilize custom components, you will often find yourself needing to pass content into your recycled component. This is where content projection comes in. Content projection allows you to pass content from a parent component to a child component. Specifically, your child component will utilize <mw-content> elements to indicate where your project content should go.

Basic Usage

Let's say we have a basic parent component and child component - a button. We want to pass the text of the button in from the parent's HTML (naturally, so we can make use of our button multiple times with different text).

Using <mw-content> without a select attribute will project all content into that element.

Using the Select Attribute on <mw-content>

If you have multiple <mw-content> elements in your child component, you can use the select attribute to specify which content goes where. The value of the select attribute is a CSS selector.

Let's expand our button example by projecting not only text, but an image element to serve as our icon. We'll use the select attribute with a basic tag name CSS selector (so no special characters in our selector other than the tag name to project).

Now, our button can accept more outside content and we can control where it gets placed (projected) in our custom button component.
