The AnglesUI Framework

In AnglesUI, you write styling for your elements in dedicated CSS files. These files must have the same base name as your HTML file, but with a .css extension.

The renderer will automatically find this file when you load your HTML into the renderer and parse, evaluate, and apply the CSS rules.

What CSS is Supported?

Not all of CSS properties and features are supported, naturally. This page serves as an exhaustive list of what is supported. In general, try to write simple CSS.

Not all properties are supported on all elements. View that element's individual documentation for what properties it supports.

General Rules and Compound Selectors

All single rules are supported as well as compound selectors.

Rule Nesting

Nested rules (a relatively new baseline CSS feature) is supported.

calc() Function

The calc() function is supported, allowing you to perform calculations to determine CSS property values. For example, the center of the screen on the X axis would be left: calc(50% - calc(800px / 2)) if your element was 800px wide.

Pseudo-States

We do not support psuedo states such as :hover, :active, :focus, etc.

:not() Selector

This selector is supported.

Container Queries

In general, we highly recommend using containers instead of media queries. They are cheaper to process but also more relevant - as they apply specifically to the containing element's side instead of the entire viewport.

Container queries are also "relatively new" in the CSS world, and can be confusing at first. Read the MDN documentation on them.
