Data Bindings

When you need your UI to display dynamic data, react to data changes, or handle the general UI state (state being its context of variables it can use), then you use data binds. More specifically, AnglesUI follows the Angular principle that any data used in a UI should be a Signal. AnglesUI implements signals for you to use.

What are Signals?

In previous iterations of web frameworks, you would provide variables into the scope of your UI. However, the only way for this framework to determine when to update the UI was to use a primitive "change detection" process which typically involved looping through all variables in the scope and checking if they had changed since the last update. This process was inefficient and could lead to performance issues, especially in larger applications. Signals, on the other hand, are a more efficient way to manage data changes. They are essentially observable values that can be subscribed to, allowing the UI to react to changes in real-time without the need for a full change detection cycle.

How to Create Signals and Pass Them to Your UI

Signal creation happens on the Lua side before you render your element. You can create the signals anywhere you like and use them however you like - so long as they get passed into the renderer when you show your UI.

Now, register them when you render the UI file.

Whenever you need to, anywhere in the code after the UI is rendered, you can update the signal's value and the UI will react to it.

Important: The text you will type to reference your signals are the indices of the table you are passing into Render().

Examples of how to call and use signals in your UI's HTML are in a few sections below on individual data binding types.

What Data Binds Are Available?

Data binding is only available in the HTML files. CSS files do not support data binding. There are four types of data binds available in AnglesUI: interpolation, style property binding, class attribute binding, and event binding.

Interpolation

This is the most basic form of data binding. It allows you to display the value of a signal directly in the UI. It's used by wrapping the signal reference in double curly braces {{}}.

Style Property Binding

All styling is done in CSS, but sometimes you need to set a style with a dynamic value. You can do this on elements using [style.propName] on the attributes. These expect to be evaluated, so you must use a data-type (such as a string, number) and now just plain text.

Class Attribute Binding

Class bindings work similarly to style property bindings, but they allow you to set the class attribute of an element dynamically. This is done using [class.className] on the attributes. These also expect to be evaluated, so you must use a data-type (such as a string, number) and now just plain text. The class will not be applied if the expression evaluates to either false or nil.

This is useful for dynamically applying classes that you have defined in your CSS.
